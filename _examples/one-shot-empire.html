Outcome

You will leave with one portable packet containing six manually completed templates: problem, audience, offer, landing copy, backlog, and experiment. The workbench reports which required fields and evidence anchors contain text; it does not generate, align, or verify their claims.

The exact prompt

The full ambition stays visible even though this lesson deliberately practices the evidence-bearing core by hand.

Here's one sentence: "[idea]". Output the entire startup:
a domain name you've verified is available, three logo concepts in
SVG, the landing page HTML, the pricing page, the first fifty GitHub
issues prioritized, a ten-slide pitch deck in markdown, and a
personalized cold DM to each of my first five customers — LinkedIn
profiles attached. Every artifact stays in voice with the others.

Build it

  1. Enter one startup sentence. The workbench repeats it as context but generates no claims.
  2. Ground the problem and audience in observations, workarounds, triggers, exclusions, and source references.
  3. Write the offer and landing copy so the promise, mechanism, proof, and call to action agree.
  4. Turn the promise into a now/next/later backlog with definitions of done.
  5. Design one falsifiable experiment with a metric, threshold, and place to record results.

Verify it

Run checks to see which required fields and evidence anchors are present. Then read across all six artifacts: the audience should experience the stated problem, the offer should answer it, landing proof should be sourced, backlog work should deliver the promise, and the experiment should be able to disprove the bet.

Failure modes

  • Fake generation: filling blanks with polished guesses is not evidence. This workbench never does that for you.
  • Artifact drift: the landing page promises something the backlog never delivers.
  • Invented proof: testimonials, traction, profiles, and domain availability must be verified outside this offline sandbox.
  • Unfalsifiable experiment: “get feedback” has no metric, threshold, or decision attached.

Remix

Add pricing, domain research, brand directions, a pitch deck, or customer outreach only after the core packet holds together. Reuse the same source sentence and evidence labels, and clearly mark every unverified field until a real check replaces the placeholder.
