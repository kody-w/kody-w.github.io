Outcome

You will leave with a Markdown review packet for one suspicious code or configuration pattern: the exact snippet, its known provenance, current owner, stated invariants, possible silent failure, attached evidence, and a transparent evidence-coverage score. The workbench does not detect AI-generated or copied code and does not infer what an author understood.

Build it

  1. Isolate the candidate. Paste only the smallest useful snippet and identify its file, service, or configuration surface.
  2. Write down what is known. Record provenance and ownership; use “unknown” instead of inventing an origin.
  3. Name the contract. State the invariant, dependent assumption, and silent failure in language another reviewer can falsify.
  4. Check only proof you have. The workbench awards points only for evidence checkboxes you explicitly select.
  5. Verify, copy, or download. Resolve the deterministic checklist and hand the packet to a human reviewer.

Verify it

Paste a tiny known snippet, leave one required field blank, and choose Verify packet. The failed field is named and focused. Fill it, check one evidence item you can support, and verify again. The same inputs and checkboxes always produce the same result and score; changing prose alone never awards evidence points.

Failure modes

  • Origin theater: a familiar-looking idiom is not proof of where this instance came from. Record “unknown” until you have a reference.
  • Checkbox inflation: checking evidence without a link, command, test, or note creates a confident-looking packet with no audit trail.
  • Danger by vibe: unfamiliar code is not automatically dangerous. Review the stated invariant and failure path, not the style.
  • Ownerless action: archaeology without a current owner becomes a report nobody can safely act on.

Remix

Duplicate the packet for several candidates and rank them in review by explicit failure impact, not by an automated “cargo-cult” label. Add team-specific evidence gates—an incident link, a version matrix, or a removal experiment—while keeping every scored item visible and manually asserted.
