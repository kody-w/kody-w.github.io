Outcome

You will leave with one downloadable, self-contained HTML artifact; an exact UTF-8 byte count against both 4,096-byte and 8,192-byte ceilings; and a short ledger explaining what the second 4KB actually purchased. The browser workbench is intentionally static: it measures, copies, and downloads your text, but never executes it.

Lesson 1 of Prompt to Proof · 25 minutes. You only need basic HTML familiarity and a modern browser.

Build it

  1. Establish the baseline. Read the starter source, select the 4KB target, and use the two comparison rows as your contract.
  2. Change one visible behavior. Add text, styling, or interaction source and watch UTF-8 bytes move. An emoji is a useful reminder that characters and bytes are not interchangeable.
  3. Make the 4KB trade. Remove ceremony before capability. Keep the dependency-syntax scan green, manually inspect its result, and do not mistake minification for architecture.
  4. Spend the second budget. Switch to 8KB and add one feature that earns its cost. Write the reason in the expansion ledger.
  5. Export the artifact. Copy it for another editor or download the exact text as an HTML file. Run it only after you have moved it into an environment you trust.

Verify it

The indicators are deterministic, not model judgments. They test whether the artifact is present, measure its encoded UTF-8 bytes, compare the same number with both fixed ceilings, conservatively flag common dependency syntax, and require an expansion note. The scan rejects quoted or unquoted remote and relative resource attributes, CSS resource functions and imports, and common runtime loaders. A green scan means only that those lexical patterns were not found; it is not proof of self-containment. A green 4KB row means bytes ≤ 4096; a green 8KB row means bytes ≤ 8192.

Copy or download the result and independently inspect it before execution. The workbench never creates a preview, injects the artifact into the page, or evaluates pasted JavaScript.

Failure modes

  • Counting characters. UTF-8 punctuation and emoji can cost multiple bytes. Trust encoded bytes, not text.length.
  • Comparing different artifacts. Keep one source in the editor while comparing both ceilings, then document the capability you add for 8KB.
  • Hiding dependencies. A tiny loader or relative asset reference is not self-contained. Treat the conservative syntax scan as a warning system, never a certificate, and inspect the source yourself.
  • Optimizing away accessibility. A smaller artifact that loses labels, keyboard behavior, or readable contrast did not buy the right thing.

Remix

Try three ceilings—1KB, 4KB, and 8KB—or assign budgets by concern: structure, styling, interaction, and accessibility. Change the canonical prompt only after you can explain the current trade. The useful remix is not merely smaller code; it is a sharper claim about which bytes make the experience meaningfully better.
