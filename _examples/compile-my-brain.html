Outcome

You will produce one exportable bundle containing a system design doc, OpenAPI spec, database schema, test plan, risk register, and team-ready Slack message. Each artifact reuses anchors you explicitly declare. This local workbench provides structure and exact-text checks; it never summarizes the transcript or fabricates a design.

Build it

  1. Keep the source visible. Paste the monologue or transcript without asking the page to interpret it.
  2. Resolve ambiguity first. Enter a canonical system name, shared entities, invariants, failure modes, and five answered clarifying questions.
  3. Load scaffolds, not answers. The guided templates contain headings and explicit placeholders only.
  4. Edit for six audiences. Shape each artifact manually, mark it complete, and preserve the same anchors everywhere.
  5. Run consistency checks. Copy or export only after every required anchor appears literally in all six artifacts.

Verify it

Load the templates, add a unique entity such as ReviewEnvelope, and verify. The workbench lists every artifact missing that exact anchor. Add it to all six, answer all five questions, mark each artifact complete, and verify again. Repeating the check with unchanged text produces the same result—there is no model call or hidden interpretation.

Failure modes

  • Template completion: headings and placeholders are scaffolding, not a finished design. Verification rejects known placeholders.
  • Vocabulary drift: renaming an entity in one artifact breaks the exact-text consistency check.
  • Unsupported certainty: a polished schema can still be fiction if its claims do not trace back to the source or an answered question.
  • Persistence surprise: browser storage is off by default. Enable it only on a trusted device and reset when finished.

Remix

Swap one scaffold for an architecture decision record, migration runbook, or launch checklist. Keep six—or change the count—but retain the same discipline: declared source, manually resolved questions, shared anchors, visible completion gates, and deterministic checks before export.
