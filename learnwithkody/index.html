What's a "vibe coding example" here?

Each entry is a project I actually shipped — usually a single-file HTML app, a GitHub-native agent, or a local-first tool — built primarily by describing what I wanted to a coding model and iterating fast. Every example links to a live demo, the source repo, and the seed prompt that got it off the ground.

Nothing here is a toy. They're production-grade, browser-runnable, and meant to be forked. The point isn't to admire the build — it's to give you a working template you can re-vibe in an afternoon.
