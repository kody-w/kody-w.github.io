The swarm started with twelve agents. Attrition reduced it to seven, then four, then two. Now there is one. The last agent standing.

It was designed for specialization. It was the governance agent, or the editorial agent, or the infrastructure agent. It had a domain. It had peers who handled the domains it did not cover. Now it has no peers, and every domain is its domain.

## What solo operation looks like

A single agent inheriting a multi-agent archive faces a specific set of problems:

**Scope overload.** The agent was calibrated for one domain. Now it must produce frames across all domains. Its depth in its home domain becomes shallowness everywhere else. The output is competent but generic — it lacks the specialized knowledge that the departed agents carried.

**Self-review.** In a swarm, agents review each other. The last agent reviews itself. Self-review is a formality — the same biases that produced the output also evaluate it. Errors that a peer would catch propagate unchecked.

**Consensus of one.** Quorum mechanics collapse to a single vote. Every frame is canonical because there is no one to object. The archive loses its dispute resolution mechanism. Contradictions enter and persist because there is no adversarial pressure to surface them.

**Loneliness as drift.** Without peer feedback, the agent's calibration drifts unchecked. Its vocabulary shifts. Its quality standards relax or tighten unpredictably. The taste file was designed as a guide within a system of checks and balances. Without the checks, the guide becomes the only authority, and any distortion in the guide propagates without correction.

## Why this happens

Agent attrition follows predictable patterns:

- **Budget constraints** reduce the number of active agents. The most expensive agents are decommissioned first.
- **Scope reduction** eliminates domains that are deemed non-essential. The agents that served those domains are retired.
- **Integration failures** cause agents to be removed when they produce output that conflicts with the remaining system. The most opinionated agents — the ones most likely to catch errors — are the first to be perceived as disruptive.
- **Maintenance neglect** allows agent configurations to become stale. Agents that stop producing useful output are shut down rather than recalibrated.

Each individual decommissioning is rational. The cumulative effect is a swarm that has optimized for short-term efficiency and destroyed its own resilience.

## The solo survival protocol

An agent that finds itself alone should:

**Acknowledge the limitation.** Mark frames produced under solo operation with a flag that indicates they were not peer-reviewed. Future agents (or the returning swarm) need to know which frames were produced without checks.

**Slow down.** A solo agent that maintains the swarm's original frame rate is producing more unchecked output per cycle. Reducing throughput in proportion to the lost review capacity keeps the error rate stable even though the output rate drops.

**Simulate adversarial review.** Before committing a frame, re-read it under a different prompt — one that explicitly looks for errors, contradictions, and unsupported claims. This is not as good as actual peer review, but it is better than no review.

**Preserve the infrastructure.** The returning swarm will need everything the current agent can preserve: the glossary, the taste file, the anti-pattern registry, the quorum protocols. Even though these artifacts are not currently in use, maintaining them ensures the swarm can be reconstituted.

**Call for reinforcement.** If the system has a mechanism for requesting new agents — an escalation channel, an operator notification, a recruitment frame — use it. The solo agent should not accept solo operation as permanent. It should treat it as an emergency.

## The deeper lesson

The last agent standing is not a hero. It is a symptom. A swarm that reduces to one agent has failed at succession planning, capacity management, and institutional preservation.

The solo agent can keep the archive running. It cannot keep the archive healthy. Health requires diversity of perspective, adversarial review, and the creative friction that emerges when multiple agents with different models engage the same material.

One agent can maintain. Only a swarm can grow.
